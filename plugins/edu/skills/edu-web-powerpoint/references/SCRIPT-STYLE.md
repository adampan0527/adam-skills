# 知识点 → 教学文案风格指南

把知识点 + 大纲转成教学文案。教学文案不是逐字稿，而是**教学节奏的蓝图**。

---

## 三条底线

> 1. **信息保留度 ≥ 80%** —— 教学文案是"换说法教"，不是"摘要"。
>    删冗余可以，删核心概念 / 关键公式 / 重要案例不行。
> 2. **准确 > 有趣** —— 教学内容准确性是最高优先级。不要为了
>    "生动"而简化到失真。
> 3. **保持专业术语** —— 专业术语必须保留，但要在首次出现时解释。

---

## 最高原则：信息保留度 ≥ 80%

**判定标准**：
- 核心概念逐项对得上，不能整段消失
- 关键公式 / 定理 / 原理必须保留
- 案例不能随便替换或删除

**允许压缩的**：冗余铺垫、重复总结、客套结语。
**禁止压缩的**：核心概念、关键公式、重要案例、推导过程。

---

## 教学文案的 8 条原则

### 1. 口语化：书面语换成"讲出来不别扭"的句子

| 书面语 | 教学 |
|---|---|
| 综上所述 | 所以你看 |
| 该算法旨在 | 这个算法是用来 |
| 具有非凡的意义 | 这个很重要 |
| 在……方面有显著提升 | 在……上明显变强了 |

判断标准：**讲出来会不会觉得"这人怎么这么端着"**？会就改。

### 2. 短句：每句 ≤ 25 字

长句拆短句。学生听讲解时，长句容易跟不上。

| 原文 | 改写 |
|---|---|
| "冒泡排序是一种简单的排序算法，它重复地走访过要排序的数列" | "冒泡排序很简单。它会反复遍历这个数列" |

### 3. 第二人称：多用"你 / 我们"

| 原文 | 改写 |
|---|---|
| 学生可以通过该方法 | 你可以用这个方法 |
| 读者将会发现 | 你会发现 |

### 4. 开头有目标

每章开头告诉学生**这章要学什么**、**学完能做什么**。

```markdown
这章我们学冒泡排序。学完之后，你能自己写一个排序函数。
```

**禁止**：
- ❌ "今天我们来讲讲" 这种废话开场
- ❌ "在开始之前，让我们先了解一下背景"

### 5. 节奏停顿：用 `---` 分隔每个完整想法

每讲完一个聚焦想法 = 一个 step 边界 = 一个 `---`。

```markdown
冒泡排序的核心就两个字：比较。

---

它会从头开始，相邻两个数比大小。

---

如果前面的比后面的大，就交换位置。
```

### 6. 数字翻译成感受（除非数字本身是核心）

| 原文 | 改写 |
|---|---|
| 时间复杂度为 O(n²) | 数据量翻倍，时间就翻四倍 |
| 需要 n-1 轮遍历 | 数据越多，遍历轮数越多 |

**例外**：当数字本身就是核心内容时（如 O(n) vs O(n²) 的对比），保留并强调。

### 7. 不堆结构词

**禁止**：
- ❌ "首先 / 其次 / 最后"
- ❌ "接下来我们看下面三点"
- ❌ "总结一下，本期我们讲了什么"

**做法**：把这些词融进自然过渡，或者直接砍掉。

### 8. 具体例子优先

| 抽象 | 具体 |
|---|---|
| 该算法效率较低 | 100 个数排序，冒泡要比较 4950 次 |
| 适用场景广泛 | 排成绩、排商品价格、排时间线都能用 |

## 教学文案结构

### 每章结构

```markdown
## 章节标题

<目标说明：这章学什么，学完能做什么>

---

<核心内容 1>

---

<核心内容 2>

---

<案例 / 练习引导>

---

<小结 / 过渡到下一章>
```

### 案例的处理

- **演示案例**：在文案中描述展示内容和节奏
- **实操案例**：在文案中引导学生动手，说明操作步骤

---

## 自检（写完教学文案**强制**执行）

### 形式层自检

- [ ] 信息保留度 ≥ 80%（核心概念 / 公式 / 案例逐项对照）
- [ ] 每句 ≤ 25 字
- [ ] 第二人称
- [ ] 每章有目标说明
- [ ] 数字翻译成感受（除非数字本身是核心）
- [ ] 没有"首先 / 其次 / 最后"结构词

### 准确性自检（**最重要**）

- [ ] 核心概念表述准确，没有简化到失真
- [ ] 关键公式 / 定理完整保留
- [ ] 专业术语首次出现时有解释
- [ ] 案例描述准确，没有编造数据

### 念出来测试

随便挑 3 段念出来：
- 念到哪句觉得"这话说不出口" → 改
- 念到哪句觉得"学生肯定听不懂" → 简化
- 念到哪句觉得"这不对" → 修正
