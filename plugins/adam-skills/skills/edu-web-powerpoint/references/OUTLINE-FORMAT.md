# `outline.md` 格式 spec

教学演示的开发计划。**用户可以直接编辑**，格式必须人类友好。

> ## ⚠️ outline 是开发计划，不是视觉规划
>
> outline 只规划**节奏 + 内容 + 信息密度 + 案例**：
>
> - 章节切分 / 每章 step 数
> - 每步屏幕内容（hero / 数据 / 标语 / 列表项）
> - 章节级**信息池**
> - **案例清单**

---

## 抽象示例（看格式）

````markdown
# Teaching Outline

> **主题**：`<theme-id>` — <一句话风格描述>
> **知识点**：<知识点标题>
> **章节数**：<N> 章 / <M> 步

---

## 1. <chapter-id> — <章节标题>（<S> steps）

**信息池**（chapter agent 按需挂角标 / 副标 / pull-quote）：
- <类型>：<内容> —— <来源 knowledge.md §X>

**开发计划**：

- step 1 — <屏幕内容>
- ...

---

## 2. <chapter-id> — ...

---

## 案例清单

| # | 名称 | 类型 | 简述 | 推荐理由 |
|---|---|---|---|---|
| 1 | ... | [实操推荐] | ... | ... |
| 2 | ... | [演示] | ... | — |

**agent 推荐**：<案例名> 作为实操练习。理由：<...>

---

## 自检（写完 outline **强制**执行）

> ⚠️ **硬性流程**：outline 写完后**必须**走自检 → 修改 → 提交。

- [ ] 每个 step 都是单一句屏幕内容描述，没有"动画"行
- [ ] 没有具体毫秒 / 秒数
- [ ] 每章首段都有「信息池」block，至少 3 条，每条带来源标注
- [ ] 章节切分符合"每章 3~8 步"经验
- [ ] 末尾「案例清单」完整，每个案例有名称 / 类型 / 简述
- [ ] 至少有一个案例被标记为 `[实操推荐]`
- [ ] 如无案例，已自动生成 2~4 个案例
````

## 字段约定

### 顶部 metadata block

| 字段 | 必填 | 说明 |
|---|---|---|
| **主题** | ✓ | Checkpoint Plan 已选定 |
| **知识点** | ✓ | 知识点标题或范围 |
| **章节数** | ✓ | `N 章 / M 步` |

### 章节标题：`## N. <id> — <title>（<S> steps）`

| 部分 | 规则 |
|---|---|
| `N` | 1-indexed 顺序 |
| `<id>` | 小写 + 连字符。会成为文件夹名 / 音频子目录 |
| `<title>` | 给人看的中文标题 |
| `<S> steps` | 该章 step 总数 |

合法 id：`intro`、`what-is-sort`、`bubble-sort`。
不合法：`WhatIsSort`、`第一章`。

### 信息池

每章独立列出从 `knowledge.md` 抽的细节集合：

```
- <类型>：<具体内容> —— <来源 knowledge.md §X / Lxx>
```

### Step 列表

```
- step N — <屏幕内容>
```

| 规则 | 原因 |
|---|---|
| `step N` 1-indexed | agent 实现时 `if (step === N - 1)` |
| **屏幕内容** | 一句话讲清楚舞台上有什么。≤ 1 行 |
| **不写动画** | 动画由 chapter agent 决定 |
| **不写时长** | 不写毫秒 / 秒数 |

### 案例清单（outline.md 末尾）

```markdown
## 案例清单

| # | 名称 | 类型 | 简述 | 推荐理由 |
|---|---|---|---|---|
| 1 | 冒泡排序可视化 | [实操推荐] | 学生输入数组，观察排序过程 | 可本地运行、步骤清晰 |
| 2 | 排序算法对比动画 | [演示] | 展示不同排序算法的速度差异 | 适合课堂展示 |
```

**类型标记**：
- `[演示]` — 做进 web-ppt
- `[实操推荐]` — 做进 case-exercise/ 文件夹

**agent 推荐**：在表格下方写一行推荐，说明哪个做实操及理由。

## 命名规则速查

| 对象 | 规则 | 示例 |
|---|---|---|
| 章节 id | 小写 + 连字符 | `intro`, `bubble-sort` |
| 章节文件夹 | `0N-<id>` | `src/chapters/01-intro/` |
| 章节组件 | PascalCase | `Intro.tsx`, `BubbleSort.tsx` |
| 章节 CSS 类前缀 | 章节缩写 | `.in-` / `.bs-` |

---

## 章节切分经验法则

- **每章 3~8 步**。少于 3 步太薄；多于 8 步学生会忘记这章在讲什么
- **每章 = 一个聚焦主题**
- **章节边界 = 大纲中明显主题切换的位置**

---

## 案例生成逻辑

### 大纲中有案例

1. 提取案例到清单
2. 给每个案例标注类型
3. 推荐一个做实操

### 大纲中无案例

1. 分析知识点核心概念
2. 生成 2~4 个案例：
   - 1 个基础案例（验证核心概念）
   - 1 个进阶案例（综合应用）
   - 0~2 个扩展案例
3. 评估可行性
4. 写入清单
5. 推荐一个做实操
