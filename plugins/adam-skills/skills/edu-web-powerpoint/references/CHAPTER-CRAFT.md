# 章节开发指引（每章开发必读）

---

## 这是教学演示，不是 PPT

正在做的是**教学网页** —— 老师点击 + 讲解 + 学生跟着看。
判断每一步做对没有，标准非常朴素：

- **不像 PPT** —— 学生感觉是在看演示，不是在看翻页幻灯
- **看起来舒服** —— 配色、字体、节奏都让人放松，不得出现大量纯文字
- **有视觉冲击** —— 画面在演事情，不只是文字堆砌

---

## 必须用 CSS / SVG / Canvas / JS 大胆绘制视觉演示

> **这是底线。**
>
> 每一章都至少要有 1~2 处"动起来的图 / 演示元素"。
> **整章只有纯文字 = 验收不过 = 回去重做。**

教学演示最强的来源 —— 学生**看见**了被讲解的东西在屏幕上演：

- 数字在递增 / 横条在生长 / 排名在交换
- 流程节点依次点亮 / 连线自绘
- 算法过程可视化
- 数据结构动态展示
- 代码执行流程动画

---

## 逐步揭示，禁止一次全展示

整页内容由**全局 `step` 计数器**驱动。设计每一步时心里要默念：
**这一步演什么，下一步演什么**。

**最重要的一条**：

> 当讲到"第一是 X、第二是 Y、第三是 Z"这种**清单 / 列表**时，
> **严禁**一个 step 把 X / Y / Z 全部 stagger 上来。

正确做法：一项 = 一个 step。X 在自己的 step 里独自亮起，讲到 Y 时
X 灰化保留 + Y 亮起。

**判断标准**：老师会一个一个念出来吗？会 → 必须逐个揭示。

---

## 内容取舍：抓重点，不要原文搬运

每个 step 屏幕上只挂这个节拍**最值得放大的 1~3 个东西**。

不要试图把知识点每个字都搬上去。那是课本阅读，不是教学演示。

---

## 双源：节奏跟教学文案，细节回知识点

> **节奏 / 顺序** 跟 **`teach-script.md` 教学文案** —— **关键顺序不能乱**。
> **画面细节 / 数据 / 案例** 回 **`knowledge.md` 知识点** 抽。

`outline.md` 已经在每章首段抽了「信息池」做参考。但**实现章节时
也必须回去翻 `knowledge.md`** —— 那里有比教学文案多得多的细节。

---

## 字体 / 配色 / 动画 / 留白 —— 教学演示基本审美

- **字号要大** —— hero 文字至少 80px 起，教室投影也能看清
- **留白要多** —— 舞台四边都要让出大留白
- **配色要舒服** —— **颜色和字体家族必须用主题 token**；
  字号 / 间距 / 时长章节按内容自由发挥
- **动画要舒服** —— 出现得干净利落，停下来不抢戏

---

## 避免 AI 味

- 紫粉 / 蓝紫对角渐变背景
- 圆角卡片 + 彩色左边框装饰
- 渐变按钮 + 大圆角药丸
- emoji 当图标用
- 假数据 / 假 logo
- 整章 N 步用同一种入场动画

缺的东西**承认缺** —— 用 placeholder 占位卡。不要用 emoji 凑。

---

## 案例的处理

### 演示案例

在对应章节中展示案例。可以是：
- 代码执行过程动画
- 数据可视化
- 交互式演示
- 对比展示

### 实操案例的引用

在 web-ppt 中可以引用实操案例：
- 展示案例的截图或简要说明
- 提示学生"这个案例在 case-exercise/ 文件夹里"
- 但不要把完整案例代码放在 web-ppt 里

---

## 代码层最小约束

### 必须用 token（换主题不破的底线）

- **颜色**：`--shell` / `--surface` / `--surface-2` / `--surface-3` /
  `--text` / `--text-2` / `--text-mute` / `--text-faint` / `--rule` /
  `--accent` / `--accent-soft` / `--accent-glow` ——
  **禁硬编码 hex / rgb / 颜色名**
- **字体家族**：`--font-display-cn` / `--font-display-en` / `--font-body`
  / `--font-mono` —— **禁硬编码字体名**

### 可硬编码 / 可 token，按内容自由

- **字号**：想要 80px 就写 80px
- **间距 / padding / margin**：按画面节奏写具体值
- **动画时长 / 缓动 / keyframe**：按动画意图写具体值
- **边框宽度 / 圆角 / 字距**：随手写

### 其它工程红线

- 不用 `setTimeout` / `setInterval` 驱动动画 —— 用 CSS keyframes
- 章节内的可交互元素加 `data-no-advance`
- 章节代码物理隔离：每章独立文件夹、独立 CSS 类前缀
- **每章必须有 `narrations.ts`**：
  - 数组长度 **=** 章节代码里 `if (step === N)` 出现的最大 N + 1
  - 每个元素 = 一个 string，该 step 的教学文本（来自 `teach-script.md`）
  - 这是**step 数的唯一真相源**

## 完工自检（写完每章**强制**执行，不可跳过）

> ⚠️ **硬性流程**：章节实现完成后**必须**走完自检 → 修复 → 汇报。

写完一章 + 在浏览器点完一遍后逐项过：

- [ ] **每章至少 1~2 处 CSS / SVG / Canvas / JS 视觉演示** —— 没有 = 回去补
- [ ] **不同 step 的主导动作不一样** —— 全章一种动画 = 回去重做
- [ ] 字号大、留白舒服、配色舒服
- [ ] 清单 / 列表逐个揭示，**1 项 = 1 step**
- [ ] 画面信息比教学文案多（回了知识点抽细节挂上来）
- [ ] 没有紫粉渐变 / 圆角彩色边框 / emoji / 假数据
- [ ] **颜色和字体家族全部走 token**
- [ ] 禁止出现小号字体、大量纯文字
- [ ] 禁止出现页眉页脚
- [ ] **`npx tsc --noEmit` 通过**
- [ ] 章节代码物理隔离：独立 CSS 类前缀
- [ ] **`narrations.ts` 存在**且长度正确
- [ ] **每条 narration 文本与 `teach-script.md` 对应段落语义一致**
- [ ] **用 Playwright 逐 step 截图验收排版** —— 启动 dev server，用 Playwright 导航到该章节每个 step 并截图，检查：元素是否重叠遮挡、文字是否溢出容器、布局是否对齐
- [ ] **无元素重叠** —— 动画元素、文字、格子之间不能有重叠遮挡（特别注意负 margin 导致的文字上侵）

任一未过 → 回去改。

## 常见反馈速查

| 反馈 | 定位 | 改法 |
|---|---|---|
| "太像 PPT" | 视觉 | 加视觉演示，减少纯文字 |
| "看不懂" | 内容 | 简化表述，加具体例子 |
| "太快 / 太慢" | 节奏 | 调整 step 拆分 |
| "不好看" | 视觉 | 调配色、字号、留白 |
| "卡顿" | 代码 | 优化动画性能 |
